# #93 — Retain conversations on `archive()` + expose archived filter + seed

Converts `FakeConversationRepository.archive(id)` from removal-from-store to flag-flip, so the upcoming Archived Discussions UI (split from #77) has something observable to render. `Conversation` gains `archived: Boolean = false`; `ConversationFilter` gains a fourth case `Archived`; one already-archived discussion is seeded so the archived stream is non-empty without any UI/test setup.

## Summary

- `Conversation` adds a defaulted `archived: Boolean = false` field. The default keeps every existing constructor + `.copy(...)` site source-compatible — no other production code needed edits.
- `ConversationFilter` becomes `enum class ConversationFilter { All, Channels, Discussions, Archived }`. Filter semantics in `FakeConversationRepository.observeConversations(filter)`: `All` = everything (literally — including archived); `Channels` = `isPromoted && !archived`; `Discussions` = `!isPromoted && !archived`; `Archived` = `archived` (regardless of `isPromoted`). The `!archived` clause was added to `Channels` symmetrically with `Discussions` so a hypothetical archived channel cannot regress the channel list, even though no flow produces one today.
- `archive(id)` replaces `records - id` with a `state.update { ... }` that copies the existing record with `conversation = conversation.copy(archived = true)`. Unknown id still throws `IllegalArgumentException` via the existing `unknown(id)` helper. Idempotent by construction: the second `archive(id)` produces an equal `Conversation` value, the underlying `StateFlow` does not re-emit, and the archived stream still contains the conversation exactly once. `lastUsedAt` is deliberately not bumped — archive is a lifecycle transition, not usage. `isPromoted` is not touched either; an archived channel (not exercised here) would route to `Archived` via the matrix above.
- Seed data: one new archived discussion `seed-discussion-archived` with `cwd = DEFAULT_SCRATCH_CWD`, `lastUsedAt = 2026-04-15T12:00:00Z` (deliberately older than the two live discussions so an archive view's natural sort puts it where you'd expect), zero messages. Added via a new `archived: Boolean = false` parameter on the existing `seedDiscussion(...)` companion helper rather than a sibling `seedArchivedDiscussion(...)` — one defaulted parameter is shorter than a duplicated helper.
- `ChannelListViewModelTest.stubRepo` gets a `ConversationFilter.Archived -> TODO("not used")` arm so its `when (filter)` stays exhaustive. `DiscussionListViewModelTest`'s stubs return `source` regardless of filter and didn't need an update.

## Patterns established

- **Add an enum case, not a parallel `observeArchivedConversations()` method.** When a new slice of an existing observable surface is needed, extend the existing typed handle (`ConversationFilter`) rather than fragmenting the interface with a sibling method. Two consumer call-sites switch on `ConversationFilter` exhaustively today (`FakeConversationRepository.observeConversations` and `ChannelListViewModelTest.stubRepo`); the compiler-enforced exhaustiveness check is the upside that pays back the extra arm. A parallel `observeArchivedConversations()` would have skipped the exhaustive check entirely. Future "show me a different slice" tickets that fit the same shape (e.g. a `Sleeping` filter, if one ever lands) should follow this precedent.
- **`.copy(field = newValue)` over targeted mutation; let `StateFlow` deduplicate.** `archive(id)` writes `record.copy(conversation = record.conversation.copy(archived = true))` unconditionally on every call. The second `archive(id)` produces a value equal to the previous map entry, so `MutableStateFlow.update { ... }` doesn't re-emit (it compares for equality) and downstream collectors don't churn. No explicit "is it already archived?" guard is needed, and adding one would obscure the intent. This generalises: any `state.update` mutator that's safe to write idempotently should be — relying on `StateFlow` equality is cheaper than an explicit guard, and no-op writes degrade to free.
- **`!archived` clauses on every non-`All`, non-`Archived` filter.** The matrix above is symmetric on purpose: only `All` shows archived items unconditionally, and only `Archived` shows them at all. A future filter case (the speculative `Sleeping` example, or any "show me a slice of live conversations") should default to including a `!archived` clause unless there's an explicit reason to surface tombstones in that view. The reverse default — "filter is on `isPromoted`, archive is orthogonal" — would have meant adding `!archived` to every filter at the call site, which is exactly the bug the symmetric default avoids.
- **Defaulted parameter on a companion helper > sibling helper for one-bit variations.** `seedDiscussion(..., archived: Boolean = false)` rather than a sibling `seedArchivedDiscussion(...)`. Two existing seed call sites stay one-line untouched; the new third call site adds `archived = true` and reads obviously. The cost (one extra parameter) is paid once at the helper; the alternative duplicates the helper body and forks future edits.

## Lessons learned

- **`MutableStateFlow.update { ... }` deduplicates on value equality; idempotency is free for `data class` `.copy(...)` mutators.** The idempotency test passes with no explicit guard in the production code because the second `archive(id)` produces an `equals`-identical map. This is a load-bearing property of the `state.update` pattern across the file, not specific to `archive` — any mutator that writes a `.copy(...)` of an existing `data class` field gets free idempotency as long as the source value already equals the target value. The brittle inverse: any mutator that uses a non-`equals`-stable type (e.g. an array, or a class without `equals` override) would silently re-emit. None today, but worth knowing if a `ByteArray` field ever lands.
- **`ConversationFilter.All` means "literally all", not "all live".** The filter matrix decision deliberately leaves `All` returning archived rows. The PO's note about bumping the `All`-size assertion from `5→6` (and `6→7`) presumes this exact semantics. The other reading — "`All` is a union of the visible filters, archived is invisible by default" — would have left `All` at 5 and required a fifth case for "everything including tombstones". That would be the worse precedent; `All` is the unfiltered baseline, and consumers that want "live only" can ask for `Channels` or `Discussions` (or both, separately).
- **`archived` on `Conversation` is not derived; `isSleeping` is.** `isSleeping` (#20) is computed at the projection seam from `currentSession.endedAt != null` because the truth lives in `Session.endedAt` and there's no separate authoritative bit. `archived` is the inverse: it's the authoritative bit (Phase 4's wire format will carry it directly), so it lives on the `Conversation` data class and the projection just passes it through. Future fields should be classified on this axis: derivable-from-existing-state → projection-stamped (don't store); independent lifecycle bit → on the data class.
