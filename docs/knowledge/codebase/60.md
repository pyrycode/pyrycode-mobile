# #60 — Scanner screen Figma polish

Second Phase 1.5 Figma-polish ticket through the pipeline (after #57 Welcome). Rewrites the #12 stub body to match locked Figma node `13:2` while preserving the `ScannerScreen(onTap)` signature and the two 412×892 light/dark previews.

## Summary

- Replaced the `Surface` + centred `Column` (rounded `surfaceVariant` box + 2dp scan line + "Tap to pair") with a `Surface` → `Column(systemBarsPadding)` of `TopAppBar` → viewport `Box(weight(1f))` → `Trouble scanning?` `TextButton`. Outer `pointerInput { detectTapGestures { onTap() } }` is unchanged; the back `IconButton` and the `TextButton` are both wired to `onTap` to satisfy AC6's "tap anywhere fires onTap" guarantee without touching `MainActivity.kt`.
- Viewport stack (back-to-front): `surfaceContainerLowest` base clipped `RoundedCornerShape(24.dp)` → both radial gradients painted inside a single `Modifier.drawBehind { drawRect(brush = Brush.radialGradient(...)); drawRect(brush = Brush.radialGradient(...)) }` (cool-blue at 30%/40%, soft-coral at 70%/70%, `radius = maxOf(size.width, size.height) * 0.7f`) → 1×1.dp-stripes-every-7.dp `Canvas` overlay → 248dp `Reticle` centered → 16dp-inset `HintCard` aligned `BottomCenter`.
- `Reticle` is a private composable: four `Corner(alignment, color = primary)` children at the corners (each a 28dp box wrapping a `28×4.dp` horizontal stub + a `4×28.dp` vertical stub, both `RoundedCornerShape(2.dp)`) plus two horizontally-padded scan-line layers — a 6dp-tall `Box(blur(12.dp, BlurredEdgeTreatment.Unbounded), background = primary.copy(alpha = 0.6f))` glow under a 2dp-tall `Box(background = primary)` crisp line, both `Modifier.align(Alignment.Center)`.
- `HintCard` wraps a single `Text(buildAnnotatedString { append("Run "); withStyle(SpanStyle(fontFamily = FontFamily.Monospace, color = colorScheme.tertiary)) { append("pyry pair") }; append(" on your pyrycode server to generate a QR code.") }, style = bodyMedium, color = onSurface.copy(alpha = 0.92f))` over a `colorScheme.scrim.copy(alpha = 0.72f)` background clipped `RoundedCornerShape(12.dp)`.
- Zero `Color(0x…)` literals; the AC5 gradient-stop carve-out is unused (token + alpha was enough). `Color.Transparent` is the only non-token `Color` (top-app-bar container + radial outer stops). Zero `TextStyle(...)` constructors — `FontFamily.Monospace` rides inside a `SpanStyle` and is a Compose-provided `FontFamily` constant, not a typography literal.

## Patterns established

- **Both radials in one `Modifier.drawBehind`, not two stacked `Box`es with `Modifier.background(brush = ...)`.** The spec floated either; the drawBehind path drops one layer (no extra `Box(Modifier.matchParentSize())` per gradient) and reads identically. It also reuses the `Modifier.drawBehind` convention #57 set for the Welcome radial — one atmospheric-overlay shape across both Figma-polish screens. The token-derived `Color` values (`blueStop`, `coralStop`) are hoisted to `val`s at the composable's top scope so the draw lambda can close over them — `MaterialTheme.colorScheme` is not addressable from `DrawScope`.
- **`Modifier.blur(..., BlurredEdgeTreatment.Unbounded)` for halo glows.** The default `BlurredEdgeTreatment.Rectangle` clips the blur to the element's bounds, which clips the halo's outer falloff. `Unbounded` lets the blur bleed past the element edge — the right shape for "glow around a thin line." Use it whenever a blur is meant to produce a halo, not a softened solid.
- **Private composables live in the same file as their sole caller.** `Reticle`, `Corner`, and `HintCard` are private to `ScannerScreen.kt`. No `ui/onboarding/components/` package created — premature factoring for composables that have one call site and a Phase 4 deletion date. The `private` modifier is the locality contract.
- **AC5 carve-out is "available, not default."** The spec offered `Color(0xFF7AB8E8).copy(alpha = 0.12f)` as a fallback if `colorScheme.primary` drifted visually for the radial stops; the implementation didn't need it. Future Figma-polish tickets: try the token first, fall back to the literal only if the preview reads off.

## Lessons learned

- **`MaterialTheme.colorScheme` is not addressable from `DrawScope` / `Canvas` lambdas.** Hoist any theme-derived `Color` to a `val` at the composable's top scope and close over it from the draw lambda. This bites three times in this file: `blueStop` + `coralStop` for `Modifier.drawBehind`, and `stripeColor` for the `Canvas` stripe loop. The error surfaces as "Unresolved reference: MaterialTheme" inside the lambda — easy to fix once you know, easy to repeat if you don't.
- **`Brush.radialGradient` in Compose is circular-only.** Figma's SVG payload for this node uses a `gradientTransform` matrix that produces an *elliptical* radial. Matching the ellipse exactly would require wrapping the draw in a `Modifier.scale(scaleX = a/b, scaleY = 1f)` Box — the architect's spec called this out as "do not implement." Parity-of-intent (haze at the right corner) > pixel-identity of the SVG matrix; precedent for future Figma tickets where Compose's brush primitives don't 1:1 the SVG ones.
- **Stripe count is a function of measured height, not a Figma-pinned constant.** AC2 specified "~105 stripes." The Figma figure is `floor(736dp / 7dp) = 105` for the locked panel height; on any other device the count differs. The `while (y <= size.height) { drawRect(...); y += 7.dp.toPx() }` loop self-terminates correctly — do not pre-allocate a `List(105) { ... }` or compute the count up-front. Density adapts to viewport size by construction.
- **`@OptIn(ExperimentalMaterial3Api::class)` belongs on the consuming composable, not the file.** `TopAppBar` is the only experimental Material 3 API in this file; a single function-level opt-in is the precise scope. A `@file:OptIn(...)` is over-broad and grants the rest of the file the same opt-in — fine when many composables in the file need it, wrong when only one does. Sized for blast radius.
- **`Modifier.blur` requires API 31+; min SDK is 33 so production is safe.** Preview environments on older API levels render the line without halo (visually degraded, not crashed) — acceptable. Don't add a manual `if (Build.VERSION.SDK_INT >= 31)` gate; the modifier handles the fallback.
