## Patterns established

- **`Canvas` over `ImageVector` for AC-enumerated primitive illustrations.** When the AC literally lists the shapes ("rect + 2 ellipses + viewfinder bridge path + error-color strike line"), reach for `Canvas` first. The threshold the architect used: count the primitives; if it's "a handful of `drawX` calls," skip `res/drawable/` and the SVG→VectorDrawable hand-conversion step (the #57 lesson). Add a vector only when the illustration has gradients, clip paths, or path counts that would balloon the `Canvas` block past readability.
- **Inner-lens is `drawCircle` *without* `style`, i.e. filled.** A 2dp-stroked tiny circle and a filled dot look near-identical at preview scale, but the filled form keeps `style` and `strokeWidth` out of the call site. The spec's open question ("stroked or filled?") resolved to filled here; future small-dot accents follow the same default.
- **Fractional canvas coordinates.** All illustration math is `w * 0.xx` / `h * 0.xx`. The illustration is sized via `Modifier.size(120.dp)` at the call site; if a future caller needs a 64dp or 200dp variant, the shape stays proportional. Don't bake absolute dp into `Canvas` blocks that have a single sized parent — let `size` be the contract.
- **`Spacer(Modifier.weight(1f))` to bottom-anchor an action stack.** The spec calls this out vs `Arrangement.SpaceBetween`. Use the explicit weight spacer when the column has more than two children and you want a clear "everything above is top-clustered, everything below the spacer is bottom-pinned" split. `SpaceBetween` distributes whitespace evenly between all children, which mis-spaces the headline/body cluster here.

## Lessons learned

- **The bridge `Path` doesn't need an explicit `close()`.** Three `lineTo` segments form a four-vertex closed shape when the last point matches the first by coordinate — and at stroke width 2dp, even a tiny gap at the join is invisible. Avoided a stray `close()` call that would have added a fifth implicit segment from the last `lineTo` back to the `moveTo`, which here coincides with the closing edge anyway. (If the bridge's first/last points ever diverge, the stroke join will gap — revisit then.)
- **Iterating Canvas coordinates is a preview-pane loop, not a math exercise.** The architect explicitly punted exact coordinates to the developer ("iterate the coordinates until it matches"). Practical loop: pick fractions from a Figma screenshot (eyeball), build, run the dark preview, side-by-side compare, tune by ±0.02 per dimension. Three iterations was enough here. Don't try to derive coordinates from `get_design_context` numeric output — that's bounding-box math for the whole frame, not per-primitive.
- **`Spacer(modifier = Modifier.weight(1f))` requires the parent to be a `Column` (or `Row`).** Obvious in retrospect, but a refactor that ever lifts the weight spacer out of the `Column` (e.g. into a wrapping `Box`) will silently lose the bottom-anchoring. If the action stack ever needs to be hoisted into a `Scaffold` `bottomBar`, the weight spacer goes away — don't carry it as a habit.
