# #210 — feat(data): `createWorkspaceFolder()` on `ConversationRepository`

Lands the write-side counterpart to [#209](./209.md): one new `suspend fun createWorkspaceFolder(name: String): String` on the [`ConversationRepository`](../features/conversation-repository.md) interface that creates a new folder under the `pyry-workspace/` prefix and registers it at position 0 of the recents stream. Plugs into the existing `recents: MutableStateFlow<List<String>>` storage and the private `bumpWorkspace(cwd)` helper added by #209 — no storage reshape, no new private state. Unblocks the Workspace Picker's **Other → Create new folder…** flow (Figma `19:44`, reached from `20:2`); the picker calls the method on dialog submit and selects the returned path. Split from #205 alongside #209.

## Summary

- **Interface gained one method with a default implementation:** `suspend fun createWorkspaceFolder(name: String): String = error("createWorkspaceFolder is not implemented for this ConversationRepository")`. Second member on the interface to ship with a default body (the first was `recentWorkspaces()` in #209). Same cascade-escape rationale — 11 anonymous `object : ConversationRepository` test fakes across 5 files (`SettingsViewModelTest`, `ArchivedDiscussionsViewModelTest`, `DiscussionListViewModelTest`, `ChannelListViewModelTest`, `ThreadViewModelTest`) inherit the default and don't need stubs.
- **Default body is `error(...)`, not a semantically-meaningful zero.** Departure from #209's `flowOf(emptyList())` shape: `recentWorkspaces()`'s default was semantically correct ("a repo that doesn't track workspace history has no recents"). `createWorkspaceFolder` returns a path `String`; there is no meaningful zero value — a throw is the right shape. `error("…")` (throws `IllegalStateException`) was preferred over `throw NotImplementedError()` because the message is clearer at the call site and matches surrounding `error(...)` use. The throw is unreachable in tests today (no fake calls the method); a future test that does will fail loudly with a clear message — that's the correct failure mode.
- **Fake override is five lines, validates then delegates:**
  ```kotlin
  override suspend fun createWorkspaceFolder(name: String): String {
      require(name.isNotBlank()) { "name must not be blank" }
      val path = "pyry-workspace/$name"
      bumpWorkspace(path)
      return path
  }
  ```
  Three behaviours in order — validate (`IllegalArgumentException` for `""`, `"  "`, `"\t\n"`), construct (literal `"pyry-workspace/$name"`, no trimming, no basename normalization — caller's contract), bump (`bumpWorkspace(path)` prepends to `recents` with dedup via the helper added by #209). The constructed path is guaranteed non-empty and cannot equal `DEFAULT_SCRATCH_CWD` (`pyry-workspace/` ≠ `~/.pyrycode/scratch` by construction), so the early-return filters inside `bumpWorkspace` never fire for this call site.
- **No `state` mutation.** Unlike the cwd-touching mutators (`createDiscussion`, `promote`, `changeWorkspace`, `startNewSession`), `createWorkspaceFolder` does not touch `MutableStateFlow<Map<String, ConversationRecord>>`. A workspace folder exists independently of any conversation in Phase 0; the persistence story for "user created a folder but no conversation uses it yet" is the recents-list-only design (see #209 spec). Phase 4 will reconcile with server-side storage.
- **Two unit tests in `FakeConversationRepositoryTest`** (JUnit 4 + `runBlocking` + `Flow.first()`, matches existing conventions). `createWorkspaceFolder_appearsAtPositionZeroOfRecents` asserts the returned path equals `"pyry-workspace/scratch-1"`, the next `recentWorkspaces().first()` emission has that path at index 0, and the list size grew by exactly one (path is new — no dedup collapse). `createWorkspaceFolder_blankName_throwsIllegalArgumentException` uses the file's existing `try { … assertTrue("expected", false) } catch (_: IllegalArgumentException) { }` shape (matches the file's adjacent style) with `"  "` as the AC-named whitespace case; `""` is also blank and would also throw — one assertion suffices because `String.isNotBlank()` collapses both.

## Patterns established

- **Default-impl shape varies with the method's return semantics.** #209 set the precedent that default bodies on `ConversationRepository` are reserved for the cascade-escape valve. This ticket refines the rule: the *form* of the default depends on whether the return type admits a semantically-meaningful zero. `flowOf(emptyList())` was correct for `recentWorkspaces()` because an empty stream is a valid "I don't track this" answer. For methods returning a non-collection value (a path `String`, a `Conversation`, a `Session`, a `Message`), there is no such zero — `error(...)` is the right shape, and `NotImplementedError` is the wrong shape because its message is less actionable. **Rule for future default bodies on this interface:** prefer a semantically-correct zero (empty `Flow`, empty list, `null`) when one exists; fall back to `error("<method> is not implemented for this ConversationRepository")` when it does not. Never use `TODO("not used")` — that's still the wrong shape for a default.
- **Validation contract on the interface KDoc, enforcement in the override.** `createWorkspaceFolder`'s KDoc documents the `IllegalArgumentException` precondition; the fake enforces it via `require(name.isNotBlank())`. A future `RemoteConversationRepository` MUST enforce the same precondition — that's the contract. Same posture the interface already takes for `sendMessage` (caller-trims, no blank-text enforcement at the data layer) and `promote` (blank-name `IllegalArgumentException`). Pattern: input preconditions belong on the interface so every implementation agrees on the contract; runtime enforcement lives in the impl.
- **A new mutator that doesn't fit the existing `state.update {}` template skips it entirely.** The four cwd-touching mutators added by #209 share the "atomic `state.update {}` then `bumpWorkspace(cwd)`" shape. `createWorkspaceFolder` doesn't create a `Conversation`, so it has nothing to write into `state` — it skips the first step and bumps directly. Lesson: don't reach for `state.update {}` if there's no `ConversationRecord` to mutate; the bump helper is its own atomic write and stands on its own. Two atomic writes is the rule when there's both a record and a workspace projection to update; one atomic write (just the bump) is the rule when there's only the projection.

## Lessons learned

- **`require(name.isNotBlank())` catches both `""` and `"  "` with one predicate.** `String.isNotBlank()` returns `false` for the empty string and any all-whitespace string (tabs, newlines included), so a single test against `"  "` covers the empty-string case by implication. Adding a second assertion for `""` would be defensive testing of Kotlin's stdlib contract — skip it. **Pattern for any future `require(x.isNotBlank())`:** one whitespace assertion is enough; `""` is structurally the same code path.
- **The fake's override delegates without any new fields or helpers because #209 left the right seam.** When #209 landed, the architect noted that the storage hook was "forward-compatible with the deferred write-side `createWorkspaceFolder(path)` follow-up (split sibling of #205) which will plug into the same `recents` via the same helper." That prediction held: the entire override fits in 5 lines, no new state, no new helpers, no changes to the existing helpers. **Lesson for the split-ticket-when-touching-the-storage shape:** when a read-side ticket lays out a `MutableStateFlow` and a bump helper that the write-side ticket can call directly, the write side stays trivial. The architectural lever is to make the write helper take the *final* value (here: the constructed path), not a *constructor* (here: the raw name) — the validation and path-building stays in the caller, the helper just touches the flow.
- **The interface KDoc names the production consumer ("the Workspace Picker is the only production consumer; test fakes never invoke this method").** That sentence is load-bearing: it documents *why* the throwing default is acceptable. Without it, a future reader might assume the default throws as a placeholder rather than as a deliberate failure mode for a method that production code never calls on a fake. **Lesson:** when shipping a default body whose contract is "this should never fire in practice," name the production consumer + name the non-consumers in the KDoc. Otherwise the next agent will read `error(...)` as TODO-shaped and replace it.
- **No `.first()` race in the test — the bump is synchronous from the caller's viewpoint.** `createWorkspaceFolder` is `suspend` for interface consistency (Phase 4 will be I/O-bound), but the Phase 0 body is non-blocking — `require`, string interpolation, and `recents.update {}` all complete before the suspend returns. So `repo.recentWorkspaces().first()` immediately after the call observes the bumped state without any `delay` / `yield` / `runCurrent` orchestration. Same property the #209 tests rely on. **Pattern:** for a Phase 0 fake whose `suspend` mutator is actually synchronous, a downstream `Flow.first()` is a deterministic read of post-mutation state; no test-timing hedge needed.
