## Patterns established

- **M3 default-everything for `CircularProgressIndicator`.** When a Figma reference shows only the active arc on `colorScheme.primary`, the safest way to satisfy "no hardcoded color literals" is to pass zero theming arguments — `color`, `trackColor`, and `strokeWidth` all default to values that match the design. If a future ticket needs to deviate, it goes through `Theme.kt` (override `progressIndicatorTheme` slots) rather than per-call-site `color = …` overrides.
- **Top-level `Text(fontFamily = …)` over `style = role.copy(fontFamily = …)` for a family-only override.** Both compile and render identically, but the top-level argument names the intent ("M3 role + family override") and keeps `TextStyle.copy(...)` out of the file — a code-review tell that often signals an inlined override. Use this shape whenever a named M3 typography role needs a single attribute swap (family, color, decoration).
- **`Arrangement.Center` vs `Spacer(weight = 1f)` is a structural choice, not a style choice.** Use `Arrangement.Center` when the screen's children are a single centered cluster (this screen). Use `Spacer(weight = 1f)` between top-clustered and bottom-pinned groups (`ScannerDeniedScreen.kt`, #61). Reaching for `SpaceBetween` when there are more than two children mis-spaces the cluster.

## Lessons learned

- **The Figma "Pair with pyrycode" `TopAppBar` is recurringly a NavHost concern, not a screen concern.** This is now the second onboarding screen (after #61 Scanner Denied) that intentionally omits the top bar shown in its Figma frame; the spec links back to #61's rationale verbatim. Future Phase 1.5 / Phase 4 catchup tickets in this nav graph: assume the top bar is host-level until the wiring ticket says otherwise. Don't re-debate it per screen.
- **Figma's 270° arc isn't a path-replication target.** The Figma file ships the indicator as a baked 3-cubic-Bézier path because the Figma API doesn't animate built-in rings — the architect's pointer ("don't reproduce the path in Compose") generalizes: when a Figma node renders a primitive as a hand-built path because Figma can't represent the animation, check whether Compose has the animated primitive native (`CircularProgressIndicator`, `LinearProgressIndicator`, `Modifier.shimmer`-style affordances) before reaching for `Canvas` / `Path`. The Canvas-illustration heuristic from #61 doesn't apply when the primitive's whole point is *motion*.
