# #121 — Scanner reticle scan-line drop-shadow glow

Surgical Figma-polish refinement to the private `Reticle()` composable in `ScannerScreen.kt`. Replaces the paired "blurred-glow `Box` + crisp-line `Box`" the #60 polish landed with a single `Canvas` that draws a 2dp line and a `BlurMaskFilter`-backed drop shadow into one drawing surface, matching Figma node `13:2`'s `28:108` "Scan line" layer (one 232×2px rect with `box-shadow: 0px 0px 12px 0px rgba(122,184,232,0.6)`).

## Summary

- Removed the two stacked `Box`es centered in the 248dp reticle (a 6dp-tall `Modifier.blur(12.dp, BlurredEdgeTreatment.Unbounded)` glow under a 2dp-tall solid-`background` line) — both bound to `colorScheme.primary` at 0.6/1.0 alpha.
- Added a single `Canvas(Modifier.align(Center).padding(horizontal = 8.dp).fillMaxWidth().height(26.dp))` that (a) draws a shadow rect through a framework `Paint` with `BlurMaskFilter(12.dp.toPx(), BlurMaskFilter.Blur.NORMAL)` via `drawIntoCanvas { it.nativeCanvas.drawRect(...) }`, then (b) draws the crisp 2dp `primary` line on top via `DrawScope.drawRect`. The 26dp Canvas height = 12dp shadow falloff + 2dp line + 12dp shadow falloff — keeps the halo's bottom/top from clipping.
- Color binding is unchanged in spirit: `primary` for the crisp line; `primary.copy(alpha = 0.6f).toArgb()` for the shadow ARGB int set on the framework paint. No `Color(0xFF7AB8E8)` literal — the Figma `rgba(122,184,232,0.6)` happens to match dark-scheme primary; the contract is token + alpha (AC4), same precedent as the radial-gradient stops in #60.

## Patterns established

- **`BlurMaskFilter` is the Compose route for a CSS-equivalent symmetric drop shadow.** Compose has no built-in modifier that renders a `box-shadow: 0 0 Npx 0 rgba(...)` analog: `Modifier.shadow(elevation, ...)` is directional (platform elevation), and `Modifier.blur(...)` on a stand-in `Box` is a stacked-element halo, not a drop shadow on a single element. The faithful primitive is `android.graphics.BlurMaskFilter(radius, BlurMaskFilter.Blur.NORMAL)` applied to a framework `Paint`, drawn into the native canvas behind the crisp shape via `drawIntoCanvas { it.nativeCanvas.drawRect(..., paint.asFrameworkPaint()) }`. `Blur.NORMAL` is the right mode for CSS `box-shadow` (symmetric Gaussian-like falloff around the shape); `SOLID` / `OUTER` / `INNER` are not analogs. Use this pattern whenever a future Figma node specifies a single-element drop shadow that the Compose primitives don't 1:1.
- **Canvas-height = source-height + 2× blur-radius.** `BlurMaskFilter` extends rendered pixels ~radius past the source rect on every side; the host `Canvas` must be tall (and/or wide) enough to contain the falloff or it clips. 26dp Canvas height = 12 + 2 + 12 for a 2dp line with a 12dp blur radius. The same rule applies horizontally — the existing `padding(horizontal = 8.dp).fillMaxWidth()` already leaves ≫12dp of margin to the reticle bounds, so horizontal clipping is structurally impossible at the 248dp reticle width.
- **Token + alpha, not raw hex, when the Figma swatch happens to match a scheme color.** The Figma shadow color `rgba(122,184,232,0.6)` is the literal value of dark-scheme `primary` at α=0.6 — but the source-of-truth is the token, not the hex. Bind to `MaterialTheme.colorScheme.primary.copy(alpha = 0.6f)` and let the light-scheme variant differ. The radial-gradient stops in #60 set this same precedent.

## Lessons learned

- **`androidx.compose.ui.graphics.Paint` is not the same as `android.graphics.Paint`.** The Compose `Paint` wraps the framework one and exposes it via `.asFrameworkPaint()`. `maskFilter`, `color` (as ARGB int), and `isAntiAlias` are set through the framework paint inside an `asFrameworkPaint().apply { ... }` block — they aren't on the Compose `Paint` surface. The Canvas `drawRect(...)` overload in `DrawScope` won't take a Compose `Paint` for arbitrary `MaskFilter` shading either; that's why the shadow rect goes through `drawIntoCanvas { it.nativeCanvas.drawRect(...) }` while the crisp line stays on `DrawScope.drawRect`. Mixing the two APIs in one Canvas is the idiomatic shape for "framework-paint shadow + Compose-paint top layer."
- **`BlurMaskFilter` requires hardware acceleration; min SDK 33 means production is in the supported envelope.** Hardware acceleration is on for all `WindowManager.LayoutParams.TYPE_APPLICATION` windows by default, and `BlurMaskFilter` renders correctly with HW accel on API 28+. No `if (Build.VERSION.SDK_INT >= ...)` gate is needed. (Compare to `Modifier.blur`'s API 31 floor from #60 — same conclusion in this project's SDK envelope, different underlying mechanism.)
- **`MaterialTheme.colorScheme` is not addressable from `DrawScope` lambdas.** Same lesson #60 recorded for the radial gradients and atmosphere stripes — hoist any theme-derived `Color` (and any `.toArgb()` ARGB int) to a `val` at the composable's top scope before the Canvas, and close over it from the draw lambda. This bites once more in this ticket: `primary` and `shadowArgb` are both hoisted at `Reticle()`'s top before the outer `Box`.
