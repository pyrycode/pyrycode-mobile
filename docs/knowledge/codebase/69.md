# #69 — Channel List: inline Recent discussions section + See all link

Replaces the bottom-of-screen `RecentDiscussionsPill` (#26) with the canonical Figma `15:8` inline section: `outlineVariant` divider, `labelLarge` "Recent discussions" header, up to three `DiscussionPreviewRow`s (title / 2-line body placeholder / monospace `workspace · time` meta), and a "See all discussions (N) →" link tinted in `colorScheme.primary`. The section is hidden entirely when no discussions exist — no orphan header. `RecentDiscussionsPill.kt` and its two strings are deleted.

## Summary

- New `DiscussionPreviewRow(conversation, onClick, modifier)` composable in `ui/conversations/components/DiscussionPreviewRow.kt` — `Column` with a title-medium title (auto-fallback to `R.string.untitled_discussion`), a 2-line body-medium placeholder (`R.string.discussion_preview_placeholder_body`), and a single label-small meta line built as an `AnnotatedString` with `SpanStyle(fontFamily = FontFamily.Monospace)` applied to the workspace fragment only.
- `formatRelativeTime` extracted from `ConversationRow.kt` into a new internal top-level helper in `ui/conversations/components/RelativeTime.kt`. Both `ConversationRow` (existing single call site) and `DiscussionPreviewRow` now consume the same-package internal function.
- File-private `discussionPreviewWorkspaceLabel(cwd, scratchLabel)` inside `DiscussionPreviewRow.kt` — distinct from `condenseWorkspace` in `ConversationRow.kt` because the rules differ (always returns a string, falls back to `"scratch"` for `DefaultScratchCwd` *and* for empty trailing segments).
- `ChannelListUiState.Loaded` and `Empty` widened with `recentDiscussions: List<Conversation>` (top-3, `.take(3)` against the upstream-sorted `Discussions` flow) alongside the existing `recentDiscussionsCount`. The `combine` body in `ChannelListViewModel` collapses the previous `discussions.map { it.size }` projection — both derived values now come from a single upstream subscription.
- Two new private composables inside `ChannelListScreen.kt`: `RecentDiscussionsSection(discussions, totalCount, onSeeAllClick, onRowClick)` (`if (discussions.isEmpty()) return`-guarded; emits divider → header → rows → see-all) and `SeeAllDiscussionsRow(totalCount, onClick)` (label + 18dp `Icons.AutoMirrored.Filled.ArrowForward`, both `colorScheme.primary`).
- The section moves *inside* the `LazyColumn` in `Loaded` as a single trailing `item(key = "recent-discussions-section")` so it scrolls together with the channels list. In `Empty`, the section renders at the top of the `Column` above the centred empty-state copy.
- Six previews on `ChannelListScreen` reshaped: `Loaded — Light` (no discussions), `Loaded + discussions — Light`, `Empty — Light` (no discussions), `Empty + discussions — Light`, `Loaded — Dark`, `Empty — Dark`. `DiscussionPreviewRow` ships three previews of its own (scratch light, scratch dark, named cwd).
- Strings: `−2` (deleted `recent_discussions_pill_label`, `cd_recent_discussions_pill`), `+6` (`recent_discussions_section_header`, `see_all_discussions_label`, `cd_see_all_discussions`, `untitled_discussion`, `discussion_preview_placeholder_body`, `discussion_preview_workspace_scratch`).
- `RecentDiscussionsPill.kt` deleted (no remaining call sites; AC verified by `grep -rn 'RecentDiscussionsPill\|recent_discussions_pill\|cd_recent_discussions' app/src/`).
- `ChannelListViewModelTest.kt` updated: every `Loaded(...)` / `Empty(...)` assertion gains the `recentDiscussions` positional field; two new tests pin the `.take(3)` slicing (`recentDiscussions_isCappedAtThree`) and the upstream-ordering contract (`recentDiscussions_orderingFollowsUpstream`).

## Patterns established

- **Section composable + early-return guard for "render this whole block only when non-empty".** `RecentDiscussionsSection`'s first line is `if (discussions.isEmpty()) return`. Call sites in both `Empty` and `Loaded` branches invoke it unconditionally; the guard owns the "no orphan header" AC. Mirrors the precedent `RecentDiscussionsPill` set in #26 (`if (count <= 0) return`) — keeping the zero-state branch inside the composable means the screen body doesn't have to fan a `discussions.isNotEmpty()` predicate across two `when` arms.
- **`AnnotatedString` + `SpanStyle(fontFamily = FontFamily.Monospace)` for a single typographic-flourish run inside an otherwise-themed `Text`.** The meta line `<workspace> · <time>` is one `Text(style = labelSmall, color = onSurfaceVariant.copy(alpha = 0.70f))` rendering an `AnnotatedString` where only the workspace fragment carries the monospace span. Don't split into three sibling `Text`s — `Row(Arrangement.spacedBy(...))` would baseline-misalign label-small at the kerning the dot-separator wants. Reach for `buildAnnotatedString { withStyle(SpanStyle(...)) { append(...) } }` whenever the *only* delta inside a single line is a font/colour/weight span.
- **A trailing `LazyColumn` `item` with a stable string key is the right shape for "non-list content that should scroll with the list".** `item(key = "recent-discussions-section") { RecentDiscussionsSection(...) }` keeps the section in the same scroll region as the channels (matching Figma's single-scroll layout) without splitting it into a sibling `Column` below the `LazyColumn`. The constant key (vs. a per-item key) is correct because the section is a single stable child; lazy diffing doesn't need to discriminate between sub-versions. Pattern carries to any "list + trailing affordance" screen.
- **Move-not-extract for cross-component utilities.** `formatRelativeTime` moved out of `ConversationRow.kt` into a peer `RelativeTime.kt` in the same `ui/conversations/components` package, kept `internal`. No new module / public API surface. The rule: when a second sibling needs an existing `private` helper, prefer relocating it into a peer file at `internal` visibility over copying or up-leveling visibility on the original definition. Don't generalise both call sites simultaneously — the function's signature was already right; only its location needed to change.
- **Explicit "do not abstract" — distinct meta-line rules between sibling components.** `ConversationRow.condenseWorkspace` returns `String?` (null for `DefaultScratchCwd` — caller omits the chip); `DiscussionPreviewRow.discussionPreviewWorkspaceLabel` returns `String` (`"scratch"` for `DefaultScratchCwd` — meta line always shows something). Both lived next to each other; the spec explicitly refused a shared `condenseWorkspace` rewrite or a shared `MetaLine` composable because the rules collide. Two near-twin helpers, each one screen away from its caller, beats one helper with a `behaviour: WorkspaceLabelBehaviour` parameter.

## Lessons learned

- **`semantics(mergeDescendants = true)` + `Modifier.clickable` order matters for "row reads as one button to TalkBack".** The See-all `Row` chain is `.fillMaxWidth().clickable(onClick = onClick).semantics(mergeDescendants = true) { contentDescription = …; role = Role.Button }.padding(...)`. Placing `.semantics` *after* `.clickable` is intentional: the merged semantics node carries the explicit `contentDescription` and `role`, and the inner label `Text` + arrow `Icon` (with `contentDescription = null`) collapse into it. The `Icon`'s null description is correct precisely because the merged parent names the action; setting a description on the icon would have TalkBack read the affordance twice.
- **`Icons.AutoMirrored.Filled.ArrowForward` is in `material-icons-core`, not `material-icons-extended`.** `AutoMirrored` is a sub-package, not a separate artifact. The existing `Icons.Default.Add` / `Icons.Default.Settings` already pull `-core` onto the classpath, so the trailing arrow lands with no catalog edit. Future Figma-anchored RTL-aware affordances should reach for `Icons.AutoMirrored.*` before considering `-extended`.
- **`.take(3)` against an upstream-sorted flow is safer than a VM-side resort.** `FakeConversationRepository.observeConversations(Discussions)` already emits `.sortedByDescending { it.lastUsedAt }`. The VM's `combine` body does `discussions.take(RECENT_DISCUSSIONS_LIMIT)` with no `.sortedByDescending` of its own. Re-sorting in the VM would have duplicated the contract and risked the two sorts disagreeing if the upstream definition drifts (e.g. Phase 4's `RemoteConversationRepository` returns server-ordered records). The new `recentDiscussions_orderingFollowsUpstream` test pins this: the VM does not re-sort, only slices. If the repository contract changes its sort key, the test fails first.
- **Widening a `data class` UiState variant forces every positional-assert test to update; budget for it.** `Empty(recentDiscussionsCount = N)` → `Empty(recentDiscussions = ..., recentDiscussionsCount = N)` rippled through six existing tests in `ChannelListViewModelTest`. Kotlin's positional-arg construction is the failure mode — none of the existing tests used named args. Plan for the cascade when widening any sealed-variant payload; the alternative (adding a separate `recentDiscussions` field reachable via a `getter`) muddies the immutable-data-class contract this codebase leans on.
- **Single-emission `combine` derives multiple values from one upstream — don't chain `.map { it.size }` separately.** The previous shape was `combine(channelsFlow, discussionsFlow.map { it.size }) { ch, count -> … }` — fine for a single derived value, but adding `recentDiscussions = discussions.take(3)` would have required either re-subscribing to `discussionsFlow` (two collections of the same cold flow, two `WhileSubscribed` lifetimes) or a `combine`-of-`combine`. Collapsing to `combine(channelsFlow, discussionsFlow) { ch, discussions -> recent = discussions.take(3); count = discussions.size; … }` is one subscription, two derived values, simpler. Reach for the un-mapped form whenever a second derived value lands.
