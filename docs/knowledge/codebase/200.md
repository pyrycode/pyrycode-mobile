# #200 — feat(ui/components): `ConnectionBanner` composable

Lands [`ConnectionBanner`](../features/connection-banner.md), the pure-UI surface that renders the [`ConnectionState`](../features/connection-state.md) sealed model from #196 with the four agreed product copy strings. `Connected` returns early with no composition (zero height); `Connecting` / `Reconnecting(secondsRemaining)` use `surfaceContainerHigh`; `Offline` uses `errorContainer` and invokes `onRetry` on tap. Pure UI — no `ThreadViewModel` touch, no `ConnectionStateSource` binding, no `ThreadScreen` edit. The follow-up wiring slice (split from #197) lifts state via `collectAsStateWithLifecycle` and threads `source.retry()` into `onRetry`.

See [`../features/connection-banner.md`](../features/connection-banner.md) for the evergreen contract.

## Patterns established

- **Resolve the per-state quartet in the public entry, delegate the render to the stateless inner.** When a composable's job is "project a sealed type to a render pair (copy + colour + handler)", do the projection in the public entry's `when` block, return early on the no-op case, and pass the resolved primitives into a private stateless inner that just paints. Same architectural shape as [`ToolCallRow`](../features/tool-call-row.md)'s public / `*Content` split, but the inner takes resolved primitives (`text`, `containerColor`, `contentColor`, `onClick`) instead of the unresolved `toolCall` — appropriate because the projection is one-shot and trivial, no need to push the `when` into the inner. Pattern applies to any future "sealed model → banner / chip / pill" composable: keep the inner stateless on `String` + `Color` primitives so previews can drive it directly without rebuilding the sealed-type fixture.
- **Always-attach `Modifier.clickable`; gate behaviour through the lambda body, not the modifier.** Routing the "ignore taps" cases through a no-op `{}` lambda assigned in the same `when` that resolves the copy keeps the `Surface` shape uniform across all visible states, keeps the M3 ripple consistent, and avoids the `Modifier.clickable(enabled = ...) vs no clickable` branching that would mean two different `Surface` constructions for "tap does nothing" vs "tap does something". Same idiom as [`ToolCallRow`](../features/tool-call-row.md)'s unconditional `Modifier.clickable(onClick = onToggle)`. Pattern applies to any future composable where a subset of states want a no-op tap target — assign the no-op lambda alongside the copy, don't fork the modifier chain.
- **Exhaustive `when` over the sealed type is the entire branch coverage** — no `else ->` arm, no defensive fallthrough. Same posture as [`MessageBubble`](../features/message-bubble.md)'s `Role`-fanout. A future fifth `ConnectionState` case is flagged here at compile time as a hard decision, not silently absorbed.
- **Previews call the public entry, not the stateless inner.** When the public entry carries load-bearing branching (here: the `Connected` zero-height return), the preview matrix must exercise the public surface to keep it as a regression artifact. Earlier components ([#131](./131.md)'s `ToolCallRow`, [#184](./184.md)'s `StreamingAssistantBody`) drove their previews from the stateless inner because the public-entry branching there was just `rememberSaveable` / `produceState` — preview-incompatible state primitives, not behaviour. Pattern: if the public entry has *behaviour* worth preview-asserting, previews call the public entry; if it has *state machinery* that previews can't drive deterministically, previews call the inner with pinned values.

## Lessons learned

- **`Surface(contentColor = ...)` is the right inheritance vehicle for the `on*` text colour, not `Text(color = ...)`.** Setting `Surface.contentColor = onErrorContainer` (and `onSurface` for the tonal cases) lets the inner `Text` resolve its colour via `LocalContentColor.current` without a per-call `color = ...` argument. The alternative (drop the `Surface.contentColor`, pass `color = contentColor` on the `Text`) compiles and renders identically today but couples the colour to a specific `Text` call site — if a future revision adds a leading `Icon` or a secondary `Text`, each would need its own `tint = ...` / `color = ...` argument. Keeping the colour at the `Surface` boundary is the M3-idiomatic shape; it's a one-keyword choice between equivalent-looking calls at the call site, but the wrong one paints the codebase into a corner the next time the row grows.
- **Literal copy strings outside `strings.xml` are fine for `ui/conversations/components/` today.** The architect spec explicitly carried the four product copy strings as Kotlin literals — same convention as [`ToolCallRow`](../features/tool-call-row.md)'s `"Input"` / `"Output"` labels, [`MessageBubble`](../features/message-bubble.md)'s preview fixture, and the inline `English-literal` strings scattered across the components package. The right time to migrate everything to `stringResource` is the codebase's first localisation pass (a coordinated change, not an ad-hoc per-component refactor); ad-hoc migration here would either (a) leave a stylistic island that the next component author has to imitate or contradict, or (b) trigger a "while you're there" cascade across the rest of `ui/conversations/components/`. Either is out of the AC.
- **The em-dash is U+2014, not `--`.** `"Offline — tap to retry"` carries the actual character per the architect spec and the AC. A `String` constant with `--` would render visually similar at small sizes but fails an exact-equals test against the spec'd copy; keep the literal correct at the source. The Kotlin file is UTF-8, the IDE preserves the character on round-trip, and `git diff` shows it as one character — no encoding gotchas observed.
- **Spec → implementation symmetry on the `secondsRemaining` interpolation.** The spec pinned `"Reconnecting in ${secondsRemaining}s"` (lowercase `s`, no space). The implementation reads `state.secondsRemaining` directly off the smart-cast `is ConnectionState.Reconnecting` arm — no intermediate `val seconds = state.secondsRemaining`, no `String.format`. The shape is `text = "Reconnecting in ${state.secondsRemaining}s"`. This is the minimum-diff shape: any helper would defend an unobserved failure mode (negative values, non-integer durations, plurals — none in AC).
