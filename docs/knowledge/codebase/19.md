# #19 — Workspace label decoration on `ConversationRow`

Second decoration layered onto `ConversationRow` (after #17 set up the row). Conversations whose `cwd` differs from a shared sentinel render a condensed workspace label next to the name; default-scratch conversations render no label. The composition shape — `Row` inside `headlineContent` with the name weighted and decorations at natural width — is the precedent #20 (sleeping-session indicator) extends.

## Summary

- New top-level `const val DefaultScratchCwd: String = "~/.pyrycode/scratch"` co-located with `Conversation` in `data/model/Conversation.kt`. Source-of-truth sentinel: UI hides the label when `cwd == DefaultScratchCwd`, fake repo seeds emit it. No new file — one line appended to the existing model file.
- `ConversationRow.headlineContent` rewrapped from a bare `Text(displayName)` to a `Row { Text(name, weight); if (label != null) Text(label) }`. Name uses `Modifier.weight(1f, fill = false)` + `maxLines = 1` + `Ellipsis` so it yields to a label that fits and ellipsizes when crowded. Label uses `labelMedium` / `onSurfaceVariant` overrides — one tier louder than the trailing timestamp's M3-default `labelSmall`, deliberately quieter than the headline's `titleMedium`.
- `condenseWorkspace(cwd)` is a `private` top-level pure function in `ConversationRow.kt` (same shape as `previewText` / `formatRelativeTime` from #17). Rule: `cwd == DefaultScratchCwd` → `null`; else strip trailing slashes, take `substringAfterLast('/')`, fall back to the trimmed whole if blank, fall back to `null` if still blank.
- Both `seedDiscussion(...)` calls in `FakeConversationRepository` (lines 282, 289) and the only literal-asserting test (`FakeConversationRepositoryTest.kt:253`) now reference `DefaultScratchCwd` instead of the raw `"~/.pyrycode/scratch"` string.
- `previewConversation(name, isPromoted)` now varies `cwd` by tier: `~/Workspace/Projects/pyrycode-mobile` when promoted, `DefaultScratchCwd` otherwise. The existing two `@Preview`s now cover both label dimensions in addition to their existing message / theme dimensions — light/with-message preview is a channel and shows the `pyrycode-mobile` label; dark/without-message preview is a discussion in scratch and shows no label. No third preview added.
- AC's only build gate is `./gradlew assembleDebug` (AC #6); passes.

- `app/src/main/java/de/pyryco/mobile/data/model/Conversation.kt` — appended top-level `const val DefaultScratchCwd` (lines 14–19) with a doc comment explaining the sentinel role.
- `app/src/main/java/de/pyryco/mobile/data/repository/FakeConversationRepository.kt` — import added; two `seedDiscussion` calls (~lines 282, 289) switched from the literal to `DefaultScratchCwd`.

- `app/src/test/java/de/pyryco/mobile/data/repository/FakeConversationRepositoryTest.kt:253` — assertion switched from `setOf("~/.pyrycode/scratch")` to `setOf(DefaultScratchCwd)`. Same scope, same coverage; keeps the source-of-truth invariant honest at the test boundary.

Total diff: 48 insertions, 5 deletions across the four files.

## Patterns established

- **Shared sentinels for `data/model/` types live as top-level `const val`s in the type's own file.** Not a new `WorkspacePaths.kt`, not inside a `companion object` introduced for the sake of one constant. The `Conversation` data class has no companion today; adding one for a single sentinel is over-engineering. Top-level `const val` is the idiomatic Kotlin choice and `data/model/` is the layer both UI and data already depend on — putting it in `data/repository/` would force UI to reach across packages; putting it in `ui/` would force the data layer to depend on UI.
- **Headline decorations compose as natural-width `Text`/`Icon` children of a `Row` inside `headlineContent`.** Not chips (`AssistChip` / `SuggestionChip` have a 32.dp min-height that visibly raises the row), not `Surface` pills, not custom-layout containers. The constraint is "no layout shift versus other rows" — only height-flat children satisfy it. This is the seam #20 (sleeping-session indicator) extends with an `Icon` rather than restarting the layout.
- **Name takes `Modifier.weight(1f, fill = false)`.** Yields remaining space *after* the decoration claims its natural width, but doesn't force itself to that full width for very short names (so the label doesn't get shoved to the right edge needlessly). Pair with `maxLines = 1` + `TextOverflow.Ellipsis` — the name ellipsizes first when long names crowd a label.
- **Decorations use `Arrangement.spacedBy(8.dp)`.** Applies only when both children render; when the label is absent (default-scratch case), the spacedBy contributes nothing and the headline reads exactly as it did pre-#19.
- **Decoration text overrides `style =` and `color =` — but only because it must read as a secondary annotation, not a second name.** Sets the precedent that the #17 "M3 defaults are the contract" rule has an explicit carve-out for decorations whose hierarchy is *intentionally* below the headline. `labelMedium` was chosen over `labelSmall` to sit one tier above the trailing timestamp's `labelSmall` — the label is annotation-of-the-name, the timestamp is independent metadata; pinning them to the same tier would make them compete for the eye.
- **Pure-function string helpers inline as `private` top-level functions in the consuming composable file.** Same shape as `previewText` and `formatRelativeTime` from #17. `condenseWorkspace` has exactly one caller today; YAGNI says don't extract for a hypothetical future "workspace picker" or "thread header badge". When a second consumer materializes, that ticket extracts.
- **Source-of-truth sentinels are imported by the test file too.** The `assertEquals(setOf(DefaultScratchCwd), …)` rewrite isn't pedantry — if a later ticket changes the constant's value, the test follows automatically rather than silently asserting a stale literal.
- **Preview seeds vary inputs by relevant dimension instead of adding a third preview.** When a new dimension layers onto an existing composable, prefer adjusting the existing previews' seed parameters so each one models a canonical case rather than padding the file with a third `@Preview`. Here `isPromoted = true` ↔ workspace-bound channel ↔ label; `isPromoted = false` ↔ scratch discussion ↔ no label. Two previews, four covered dimensions (theme × message × label × promotion).

## Lessons learned

- **`if (workspaceLabel != null) { Text(...) }` inside a `Row` body is fine — distinct from the `supportingContent = { if (x) Text(...) }` trap from #17.** That earlier trap was about `ListItem`'s height collapse depending on the *slot lambda* being null, not on whether the lambda emitted anything. Inside a `Row` body there's no slot-null signal; `if (...) emit Text else emit nothing` is just normal Compose conditional emission and `Arrangement.spacedBy` does the right thing for the resulting child count. Two superficially similar-looking patterns; different layout contracts.
- **`Modifier.weight(1f, fill = false)` is the right shape for "yield remaining space but don't force-fill it".** `fill = true` (the default) would expand a short name to push the label to the trailing edge of the headline, leaving an awkward gap. `fill = false` lets the name take its natural width unless competition forces it to ellipsize. Worth pinning because the bare `Modifier.weight(1f)` reflex is wrong in this position.
- **Tilde-prefixed paths split on `/` like any other path.** `condenseWorkspace("~/code")` returns `"code"`, *not* `"~/code"` — the `~` is just the first segment. Intentional and matches the AC's worked example (`~/Workspace/Projects/KitchenClaw/` → `KitchenClaw`), but it's the kind of edge case that reads wrong at first glance. The bare-`~` case (no `/`) falls into the second fallback and renders as `"~"`, which reads as "home" sensibly.
- **The `ifBlank` cascade chains in priority order top-to-bottom: segment → trimmed-whole → null.** `String.ifBlank` is the standard Kotlin null-coalescing idiom for "use this if not blank, else fall back". Reaching for `Regex` for a "strip slashes, take last segment, blank-fallback" rule would be a code smell. Pinning the idiom so the next workspace-string helper doesn't import `kotlin.text.Regex` reflexively.
- **The preview path also changed (`/Users/dev/projects/pyrycode-mobile` → `~/Workspace/Projects/pyrycode-mobile`) so previews match seed-data convention.** The shortened label is the same either way (`pyrycode-mobile`); the change is pure preview/seed visual consistency — `FakeConversationRepository` channels use tilde-prefixed paths and the preview should too, so what an engineer sees in the IDE matches what they'd see at runtime.
- **`createDiscussion(workspace: String?)` writes `cwd = workspace ?: ""` — an empty string, not `DefaultScratchCwd`.** This works correctly at the UI without a data-layer change: `condenseWorkspace("")` falls through the blank cascade to `null` and renders no label. Aligning the writer with the sentinel is deliberately deferred — AC #1 only requires *a* source of truth for the constant, not that every emitter use it. If someone files a follow-up, the fix is one line in `createDiscussion`.
