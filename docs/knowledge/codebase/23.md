# #23 ‚Äî Channel list empty state

Swaps the `ChannelListUiState.Empty` body copy from the placeholder `"No channels yet"` to the call-to-action `"Tap + to start a conversation"` (loaded from a new `R.string.channel_list_empty` string resource), and adds a second `@Preview` so the Empty state is renderable in Android Studio alongside the existing Loaded preview. No VM, repository, or navigation changes.

## Summary

- `ChannelListUiState.Empty` now renders `CenteredText(stringResource(R.string.channel_list_empty), bodyModifier)` instead of the bare literal. The `CenteredText` helper, the surrounding `Scaffold`/`TopAppBar`/FAB chrome, and the other three `when (state)` branches (`Loading`, `Error`, `Loaded`) are untouched ‚Äî touch was scoped to the single Empty arm.
- New `<string name="channel_list_empty">Tap + to start a conversation</string>` in `app/src/main/res/values/strings.xml`. User-facing copy, not a `cd_` content description; follows a `<screen>_<role>` shape that future user-facing strings should mirror.
- New `@Preview(name = "Empty ‚Äî Light", showBackground = true, widthDp = 412)` named `ChannelListScreenEmptyPreview`. Wraps `ChannelListScreen(state = ChannelListUiState.Empty, onEvent = {})` in `PyrycodeMobileTheme(darkTheme = false)`. No sample data required ‚Äî `Empty` is a `data object`. Renders the TopAppBar (from #21) above the centred copy and the FAB (from #22) at `FabPosition.End`, visually confirming that the "+" in the copy lines up with the on-screen affordance.
- **VM trigger was NOT widened** to Plan.md's "no channels AND no discussions". The architect's call (documented in the spec's "Decision" section): widening would introduce a `channels=‚àÖ, discussions=non-empty` case with no clean current `ChannelListUiState` representation (`Loaded(emptyList())` would render a blank `LazyColumn` ‚Äî strictly worse than the new copy). `ChannelListViewModel.kt` and `ChannelListViewModelTest.kt` are unchanged; the AC's "If the architect widens‚Ä¶" conditional did not fire. The existing `empty_whenSourceEmitsEmptyList` test continues to pin the channels-only trigger semantics.
- `./gradlew assembleDebug` passes. No new tests ‚Äî the change is a pure UI presentation swap verified via the new Preview.

## Patterns established

- **User-facing copy uses a `<screen>_<role>` string-resource name, not a `cd_` prefix.** `channel_list_empty` ‚Äî first user-visible (non-a11y-description) string in the project; `cd_open_settings` / `cd_new_discussion` keep the `cd_` prefix because they are TalkBack content descriptions, not on-screen copy. The naming split scales: the next on-screen string on the same screen would be `channel_list_<role>`, on the next screen `<that_screen>_<role>`. Don't reuse `cd_*` keys for visible copy even when the text is the same.
- **One `@Preview` per discrete `UiState` variant when the variant has a no-data shape.** `data object` states (`Loading`, `Empty`, `Error` once it lands a designed visual) deserve their own `@Preview` with `state = <Variant>, onEvent = {}` ‚Äî the cost is ~10 lines and the design surface is genuinely different. Don't introduce a `@PreviewParameterProvider` to parameterise across states until at least three of them have non-trivial bodies; explicit per-state previews stay greppable and survive copy tweaks better than a parameterised supplier.

## Lessons learned

- **Decision rule for "widen the trigger now vs defer": does the deferred case have a faithful current representation?** Widening to `channels=‚àÖ ‚àß discussions=‚àÖ` would have produced a new case (`channels=‚àÖ, discussions=non-empty`) with no clean UI shape under the existing `ChannelListUiState`. The choice to defer is principled, not lazy: the current channels-only trigger maps `Loaded(channels)` to "has channels" and `Empty` to "no channels", and the new copy reads correctly under both deferred sub-cases. Future architects facing a similar "should we widen the input" call should ask the same question first ‚Äî if the wider domain has no representable output, defer until the output is also designed.
- **`Empty` as a `data object` is the friction-free preview shape.** No `Clock.System.now()`, no sample `Conversation` list, no theme override ‚Äî `state = ChannelListUiState.Empty, onEvent = {}` is the whole preview body. The cost of *not* keeping these placeholder states as `data object`s shows up here: a hypothetical `Empty(reason: EmptyReason)` would force the preview to invent a reason, locking visual-only previews into product-state coupling. When extending Empty later, prefer adding a *sibling* `data class` variant (`EmptyWithDiscussions(count: Int)`) and leaving the bare `Empty` `data object` intact for the universal-empty case.
- **The "+" in the copy is a referent, not a token; no string interpolation.** Plan.md's wording is verbatim `"Tap + to start a conversation"` with a literal plus sign ‚Äî not a placeholder for a unicode glyph or icon font. The FAB renders `Icons.Default.Add` whose visual shape *is* a plus; the user reads the printed `+` in the copy and pattern-matches against the on-screen FAB. Don't be tempted to swap the literal `+` for a string-builder template (`Tap %1$s to start a conversation` with the glyph injected at composition time) ‚Äî the literal survives translation as a typographic glyph and avoids a runtime substitution path for no gain.
- **`stringResource` and `R` were already imported in `ChannelListScreen.kt`.** No additive imports needed ‚Äî `R.string.app_name` (TopAppBar title, project-init), `R.string.cd_open_settings` (#21), and `R.string.cd_new_discussion` (#22) all already trigger the same imports. The third user-facing string on this screen was a zero-import-churn change; future strings on the same file remain additive-only.
