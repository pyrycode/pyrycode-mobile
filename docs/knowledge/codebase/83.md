# #83 — Spotless + ktlint formatter wiring

Wires the Spotless Gradle plugin (`com.diffplug.spotless` 7.0.4) at the root build script with three formatter blocks — `kotlin { ktlint(1.5.0) }` for `**/*.kt`, `kotlinGradle { ktlint(1.5.0) }` for `**/*.gradle.kts`, and a built-in `format("misc")` (`trimTrailingWhitespace` + `endWithNewline`) for `**/*.{md,json,yml,yaml}`. ktlint defaults are enforced verbatim with one documented `.editorconfig` exemption (`ktlint_function_naming_ignore_when_annotated_with = Composable`); the auto-fix pass commits alongside the wiring and reformats ~35 source files. `./gradlew spotlessCheck` is auto-wired into `check`.

## Summary

- `build.gradle.kts` (root) — Spotless plugin alias added without `apply false` (the only root-level plugin); `spotless { … }` block configures three target sets. `ktlint(libs.versions.ktlint.get())` threads the catalog version through Spotless's `ktlint()` accessor so the ktlint runtime is pinned in one place.
- `gradle/libs.versions.toml` — `ktlint = "1.5.0"` and `spotless = "7.0.4"` added under `[versions]`; `spotless = { id = "com.diffplug.spotless", version.ref = "spotless" }` under `[plugins]`. Both placed in the existing alphabetical ordering at the bottom of each section.
- `.editorconfig` (new, repo root) — four lines: `root = true` + `[*.{kt,kts}]` block carrying *only* `ktlint_function_naming_ignore_when_annotated_with = Composable`. The Compose-PascalCase exemption is the sole deviation from ktlint defaults.
- `app/build.gradle.kts` — no formatter wiring (Spotless is root-only). The auto-fix touched two cosmetic blocks (the `compileSdk { release(36) { … } }` chain wrapped onto its own line, a trailing comma added to `proguardFiles(...)`) and added the missing final newline.
- Property-naming renames (Kotlin official style for `const val`):
  - `MainActivity.kt` `Routes` object — `Welcome` / `Scanner` / `ChannelList` / `DiscussionList` / `ConversationThread` / `Settings` / `License` → `WELCOME` / `SCANNER` / `CHANNEL_LIST` / `DISCUSSION_LIST` / `CONVERSATION_THREAD` / `SETTINGS` / `LICENSE`; the file-private `SetupUrl` → `SETUP_URL`. All references inside `MainActivity.kt` flipped in the same commit.
  - `data/model/Conversation.kt:20` — top-level `const val DefaultScratchCwd` → `DEFAULT_SCRATCH_CWD`. Single import site (`FakeConversationRepository.kt`) updated.
- Wildcard-import + template-file cleanup — `app/src/test/java/de/pyryco/mobile/ExampleUnitTest.kt` and `app/src/androidTest/java/de/pyryco/mobile/ExampleInstrumentedTest.kt` deleted outright (Android Studio scaffolding; no callers, replaced long ago by real test suites under `data/`, `ui/`).
- `FakeConversationRepository.kt` — four seed-data string literals at the apartment-lease seed shortened (e.g. `"Got it — I'll add a calendar entry for Friday afternoon."` → `"Got it — I'll add a calendar."`) so they fit ktlint's default 140-char `max-line-length` once the surrounding indentation grew during reformat. Pure copy edit; no behavioural change.
- Mechanical churn across the rest of `app/src/` from ktlint defaults: import ordering, trailing commas in argument lists, expression-body line wrapping (`val state = MutableStateFlow(...)` → `val state =\n    MutableStateFlow(...)`), chained `.filter { … }.sortedByDescending { … }` reflowed onto the prior line's closing brace, etc. `Theme.kt` had the largest diff (501 lines reformatted) — colour scheme tables reflowed but values are byte-identical.
- Verified `./gradlew spotlessCheck` and `./gradlew check` both green after the auto-fix pass; the latter triggers `spotlessCheck` automatically (no manual `dependsOn` needed — Spotless 6.x+ auto-wires into the `check` lifecycle).

## Patterns established

- **Spotless lives at the root `build.gradle.kts`, never in `app/`.** Single-module today, future modules included for free via `**/*.kt` globs. Non-Kotlin formats (`docs/**/*.md`, repo-root `*.md`) live outside `app/` and would be unreachable from a module-scoped block. When a second module appears, do not move Spotless — the root globs already walk new sources.
- **Versions for both Spotless and ktlint are pinned via the catalog; `ktlint()` is invoked as `ktlint(libs.versions.ktlint.get())`, never bare `ktlint()` and never with a literal version string.** One edit point for the ktlint runtime version. Spotless's `ktlint()` accessor is overloaded — the no-arg form floats, the string form pins; always pass the catalog string.
- **`.editorconfig` carries exactly one ktlint property and `root = true`.** Anything else needs its own ticket and an evidence-based justification; the file is deliberately a fence around the ktlint defaults, not a configuration surface. The Compose `function-naming` exemption is the *only* documented deviation — ktlint's upstream-provided escape hatch for a framework-mandated convention. Future tickets that want to relax another rule add the exemption with their own justification, not a "while we're here" line in this file.
- **Top-level `const val` properties follow `SCREAMING_SNAKE_CASE` (Kotlin official style).** `Routes.WELCOME`, `DEFAULT_SCRATCH_CWD`, `SETUP_URL`, etc. The pre-existing `STOP_TIMEOUT_MILLIS` / `RECENT_DISCUSSIONS_LIMIT` / `FALLBACK_CHAR` / `INITIALS_LENGTH` constants were already correct; this ticket brought the outliers into line. Do not exempt `property-naming` — the rename surface is the kind of drift the gate is for.
- **`@Composable` functions stay PascalCase.** This is the only naming convention this repo intentionally diverges from on; the `.editorconfig` exemption is what makes ktlint stop complaining. Do not rename composables to camelCase under any circumstance — the entire Compose ecosystem (compiler diagnostics, Google samples, AOSP) assumes PascalCase.
- **Wildcard imports are forbidden.** ktlint default is `no-wildcard-imports`; the two Android Studio template files (`ExampleUnitTest.kt`, `ExampleInstrumentedTest.kt`) were deleted rather than fixed because they had zero callers and no production value. Going forward: expand wildcards to the specific imports actually used; do not suppress.
- **`format("misc")` for Markdown / JSON / YAML uses Spotless's built-in trim-and-newline only — no Prettier, no Jackson, no Node.js dependency.** The AC requires *a* formatter configured, not structural reformatting. If a future ticket wants pretty-printed JSON or Prettier-formatted Markdown, it can swap the block; the floor today is "no trailing whitespace, every file ends with a newline."
- **Auto-fix is not optional.** `./gradlew spotlessApply` is the only acceptable way to resolve a Spotless violation; hand-editing files to "improve" the auto-fix output defeats the determinism the gate is supposed to provide. The exceptions are violations ktlint refuses to auto-fix (line length, property naming) — those resolve with a real edit, but the edit is mechanical (rename, extract local, shorten string), never a stylistic judgement call.

## Lessons learned

- **ktlint's `standard:function-naming` rule is incompatible with Compose out of the box and the upstream-provided fix is `.editorconfig`-only.** The first developer attempt followed the original spec ("no `.editorconfig`") and immediately hit 11 violations on every existing `@Composable` in the repo. ktlint upstream introduced `ktlint_function_naming_ignore_when_annotated_with = Composable` specifically for this — there is no Spotless-block-level equivalent, no `@Suppress` annotation that scales, and no Gradle DSL hook. The `.editorconfig` line is the canonical answer; apologies are not required.
- **`isMinifyEnabled` / `isShrinkResources` / R8 do not interact with Spotless.** Spotless runs on source, ktlint runs on source, both run before `assemble`. The #81 R8 baseline stays untouched; nothing in the formatter wiring affects shrinkage or release-build behaviour. Verified by running `./gradlew check` post-wiring.
- **Spotless's `ktlint()` accessor accepts a String version and has done since well before 7.0.** The architecture doc's worry about version-resolution edge cases (`com.pinterest.ktlint:ktlint-cli` vs the artifact Spotless consumes) didn't materialise — `7.0.4` resolved `1.5.0` cleanly via `gradlePluginPortal()`. If a future bump fails to resolve, check the artifact identifier on the Spotless changelog before assuming the catalog is wrong.
- **The seed-data line-length violation in `FakeConversationRepository.kt` ended up resolved by shortening the strings, not by extracting a `val` or splitting the literal.** The architect's spec offered both options; the chosen path produced less diff churn. The user-facing implication is that the apartment-lease seed messages now read terser (`"I'll add a calendar."` rather than `"I'll add a calendar entry for Friday afternoon."`) — adjust if a future seed-data ticket prefers the longer copy. The data layer itself was not affected.
- **`./gradlew check` auto-wires `spotlessCheck` without an explicit `dependsOn`.** The architecture doc's defensive note ("if the auto-wire does not happen…") was unneeded here. Spotless 6.x+ has done this for years; verify by running `./gradlew check --dry-run` and grepping for `spotlessCheck` in the task list before adding manual hooks.
- **Preview Composables also count as `@Composable` for the function-naming exemption.** The `.editorconfig` line covers `WelcomeScreenPreview`, `ChannelListLoadedDarkPreview`, etc. — they're annotated `@Preview` *and* `@Composable`, and ktlint matches on the `Composable` annotation specifically. No additional exemption needed for previews.
- **The `format("misc")` glob for `**/*.md` reaches `docs/`, `docs/knowledge/`, repo-root `README.md`, and `CLAUDE.md` — the auto-fix changes Markdown files that were authored without a final newline.** This includes per-ticket files in `docs/knowledge/codebase/` written by past documentation runs. Future documentation work that lands a new `.md` file should end with a newline as a matter of habit; failing that, the next `spotlessApply` will add one (and credit it to whoever runs the formatter, not the original author).
