# #17 — `ConversationRow` composable

First UI primitive consuming the `Conversation` + `Message` model. One stateless Material 3 `ListItem`-based row, two `@Preview`s (light with-message, dark without), one inline private relative-time helper. No consumers wired this ticket — #18 (Channel list) is the first.

## Summary

- One new file: `app/src/main/java/de/pyryco/mobile/ui/conversations/components/ConversationRow.kt` (140 lines). New package directory `ui/conversations/components/` created implicitly via the package declaration.
- Public surface is the architect's verbatim signature: `(Conversation, Message?, () -> Unit, Modifier = Modifier)`. `modifier` is included (unlike `WelcomeScreen` where it was rejected) because the row sits inside a `LazyColumn` in #18 and consumers pass `Modifier.fillMaxWidth()` / `animateItemPlacement()`.
- Layout is a single M3 `ListItem` — `headlineContent` / `supportingContent` / `trailingContent`. No custom `Row`/`Column` tree, no `style =`, no `colors =`. The M3 default slot styling (`titleMedium` / `bodyMedium` / `labelSmall`, `onSurface` / `onSurfaceVariant`) is the contract.
- `supportingContent` is passed as `lastMessage?.let { … {} }` so the parameter is genuinely `null` when there is no message. That's what triggers `ListItem`'s built-in height collapse from ~72dp (two-line) to ~56dp (one-line) — the "no layout shift beyond the missing preview line" AC is satisfied for free.
- Auto-name fallback (`name?.takeIf { it.isNotBlank() } ?: if (isPromoted) "Untitled channel" else "Untitled discussion"`) depends only on stable fields. Treats `null` / `""` / whitespace-only equivalently.
- Preview text whitespace-normalized via `Regex("\\s+")` → single space + trim, then `maxLines = 2` + `TextOverflow.Ellipsis`.
- Relative-time helper inline as a `private` top-level function with `now: Instant = Clock.System.now()` and `timeZone: TimeZone = TimeZone.currentSystemDefault()` defaults. Buckets: `< 1m` → `"just now"`, `< 1h` → `"Nm ago"`, `< 24h` → `"Nh ago"`, `< 48h` → `"Yesterday"`, `< 7d` → `"Nd ago"`, same year → `"May 5"`, otherwise → `"May 5, 2025"`. Negative delta (clock skew) also routes to `"just now"`. Uses `kotlinx.datetime` `LocalDate.Format` DSL with `MonthNames.ENGLISH_ABBREVIATED`.
- Two `@Preview` composables, both `private`, both wrapped in `PyrycodeMobileTheme`: with-message in light mode, without-message in dark mode (`Configuration.UI_MODE_NIGHT_YES`). Seeds via two `private` top-level helpers `previewConversation(name, isPromoted)` / `previewMessage(content)` — local to the file, not extracted.
- No new tests. AC's only build gate is `./gradlew assembleDebug` (AC #7) and it passes.

## Patterns established

- **Reusable list rows accept `modifier: Modifier = Modifier`.** Distinct from full-screen composables like `WelcomeScreen` where the modifier was rejected. The discriminator is consumer expectations: a `LazyColumn` item needs `Modifier.fillMaxWidth()` / `animateItemPlacement()`, a screen does not. Rows = yes, screens = no.
- **`ListItem` is the default M3 surface for a name/preview/trailing layout.** Don't reach for a custom `Row { Column { … }; … }` when `headlineContent` / `supportingContent` / `trailingContent` slots map to the requirement. The built-in slot typography, color tokens, accessibility role, and conditional height collapse together cost zero lines.
- **Optional supporting slots are gated by `lastMessage?.let { … {} }`, not `{ if (x) Text(...) }`.** The former passes a genuinely-`null` slot to `ListItem` (height collapses); the latter passes an always-non-null lambda that emits nothing (height reserved, AC violated). The distinction is invisible from the `Text` output but load-bearing for the layout contract.
- **Relative-time helpers take `now: Instant = Clock.System.now()` as a default parameter.** Lets previews and any future unit test inject deterministic values without bumping the helper's visibility or threading the clock through the public surface. Same shape for any future "format X relative to now" helper.
- **`MonthNames.ENGLISH_ABBREVIATED` + `LocalDate.Format` DSL is the in-project idiom for human-readable absolute dates.** Top-level `private val sameYearFormat = LocalDate.Format { … }` declarations are cheap to declare and avoid re-parsing the format on every call. Don't reach for `SimpleDateFormat` or `java.time.format.DateTimeFormatter` — both would punch through [ADR 0001](../decisions/0001-kotlinx-datetime-for-data-layer.md).
- **"Yesterday" is a 48-hour duration bucket, not a calendar-day comparison.** Calendar-day "yesterday" flips at midnight (`23:59:30` → `"Yesterday"` 31 seconds later). The duration shape is what users actually expect and what Slack / Discord / iMessage do in practice.
- **Preview seeds (`previewConversation`, `previewMessage`) live as `private` top-level functions in the same file as the previews.** Not a shared `PreviewData.kt`, not a test-fixtures module. Each consumer file defines its own seeds — they're 20 lines, hyper-local, and #18 will define its own seeds for the list cases.

## Lessons learned

- **`supportingContent = { if (x) Text(...) else null }` does NOT collapse `ListItem`'s height.** The lambda is non-null from `ListItem`'s perspective, so it reserves the two-line slot even when the lambda emits nothing. The only thing that collapses the row is `supportingContent = null` itself — which means the conditional has to be on the `() -> Unit?` parameter, not inside it. The spec called this out explicitly, but it's the kind of subtle layout contract that's easy to violate during a later "tidy" pass; pinning it here.
- **Default-`Clock.System.now()` parameters mean the previews aren't pixel-stable across IDE preview runs.** The seed sets `lastUsedAt = Clock.System.now() - 2.hours`, which always renders as `"2h ago"` because both sides of the subtraction use the same `Clock.System.now()` reading taken seconds apart — but a unit test that asserted on the rendered text would need to inject `now`, not rely on this coincidence. Documented because the next person reading the previews might assume the timestamps are computed from a frozen `Instant.parse(...)` literal (the architect's spec showed that variant); they're not. The deviation is intentional — pinning a literal date would have aged into a `"6 months ago"`-class string by the time the previews are next opened.
- **Architect's seeded `Instant.parse("2026-05-12T10:00:00Z") - 2.hours` was swapped for `Clock.System.now() - 2.hours` during implementation.** Both produce a "recent" timestamp; the latter ages gracefully. If a deterministic preview is ever needed (e.g. for screenshot diffing), the architect's literal shape is the right starting point — but it requires keeping the literal close to "today" or the rendered text drifts into the absolute-date buckets.
- **The `private val sameYearFormat` / `private val crossYearFormat` declarations sit at top level, not inside `formatRelativeTime`.** Inlining the `LocalDate.Format { … }` block inside the function body would re-allocate the formatter on every call. Top-level `val` is once-per-classloader; matters when the row recomposes inside a `LazyColumn` scroll.
- **`Modifier.clickable(onClick = onClick)` already sets `Role.Button` and the ripple — no `Modifier.semantics { ... }` needed.** TalkBack reads the `Text` children naturally. Don't be tempted to add a `contentDescription` "for accessibility"; it would shadow the rendered text and make the row read worse, not better.
