# #154 ‚Äî Channel row matches Figma 15:8 (drop workspace label, sleep dot after name)

Walks `ConversationRow` back to the single-line "avatar + name + sleep-dot + timestamp" shape Figma node `15:8` specifies. The inline workspace label (#19), the supporting-content message preview (#17), and the now-dead `lastMessage: Message?` parameter are all gone; the sleeping-session dot (#20) moves from before the name to after it. Affects every screen that renders the shared row primitive ‚Äî channels, discussions, and archived discussions ‚Äî by design.

## Summary

- `ConversationRow` signature drops `lastMessage: Message?`. The supporting-content slot, the `condenseWorkspace(cwd)` lookup + inline workspace `Text`, the `previewText` whitespace normalizer, and the `previewMessage` preview seed all delete with it.
- Inside `headlineContent`'s `Row`, the order of the two children flips: the name `Text` is now first, the 8.dp sleep-dot `Box` (rendered only when `conversation.isSleeping`) is now second. The `Row` keeps `verticalAlignment = CenterVertically` + `horizontalArrangement = spacedBy(8.dp)`; the gap matches Figma node `15:68`'s inner-frame spacing.
- Two `@Preview` composables rename to reflect the surviving states the AC asks for: `ConversationRowWithMessagePreview` ‚Üí `ConversationRowPromotedPreview` (named promoted channel, light), `ConversationRowWithoutMessagePreview` ‚Üí `ConversationRowUntitledSleepingPreview` (untitled discussion + sleeping dot, dark). Both drop their `lastMessage = ‚Ä¶` argument.
- Five production call sites lose the `lastMessage = null` argument: `ChannelListScreen` (1), `DiscussionListScreen` (1 production + 2 inside `@Preview`s), `ArchivedDiscussionsScreen` (1). No other changes at any call site.
- Two imports removed from `ConversationRow.kt`: `de.pyryco.mobile.data.model.Message`, `de.pyryco.mobile.data.model.Role`. `DEFAULT_SCRATCH_CWD` stays ‚Äî still used by the `previewConversation` seed at the bottom of the file.

## Patterns established

- **Figma-locked shared rows trump per-screen decorations.** `ConversationRow` accreted the workspace label in #19 and the leading sleep dot in #20 before Figma `15:8` was canonical; once the design source landed (#68 pulled in the leading avatar against the same node), both decorations were redundant against the locked spec. The rule going forward: when a Figma node is the canonical spec for a shared row composable, that one frame governs every screen that renders the row ‚Äî don't add per-screen "make this row look slightly different" arguments. The de-emphasis alpha `DiscussionListScreen` and `ArchivedDiscussionsScreen` apply via the *outer* `Modifier.alpha(0.65f)` (#24) is the right shape: a screen-level decorator on top of one canonical row, not a parameter inside the row.
- **Walk back parameters when their only call-site value is the null sentinel.** All five production call sites already passed `lastMessage = null` ‚Äî the parameter had no live consumers anywhere in Phase 0 (no per-`Conversation` last-message projection exists in the data layer). The two-line cost of "pass `null` everywhere" was already invisible; the right move was to delete the parameter rather than leave it as a future-use placeholder. If a real consumer materializes, that ticket re-adds the parameter at the same shape ‚Äî Kotlin's keyword arguments make the round-trip cheap.
- **Rename `@Preview` composables when they outlive their original distinction.** "With message" / "Without message" stopped being the meaningful axis once `lastMessage` was gone. Rename to the actual axis that survives (promoted-named-channel vs. untitled-sleeping-discussion). Future readers should see preview names that describe the state matrix they cover, not the parameter that was removed two tickets ago.

## Lessons learned

- **`DEFAULT_SCRATCH_CWD` stays imported even though `condenseWorkspace` is gone.** The spec's "delete the `DEFAULT_SCRATCH_CWD` import" note was a misread ‚Äî the import is also consumed by the `previewConversation` seed (line 115) for the unpromoted-discussion preview branch. `./gradlew lint` would have caught a wrongful removal as an unresolved reference, but the lint pass was the safety net; the right read is "check every reference before declaring an import dead." The Compose-tooling-preview seeds count as references for this purpose.
- **`Row` child-order edits don't need wrapper changes when the spacing is uniform.** `Arrangement.spacedBy(8.dp)` applies the gap between *every* pair of rendered children regardless of order, so flipping the name and the sleep dot did not require any parent-modifier edit. If the spec ever wanted asymmetric spacing (e.g. 4.dp between name+dot and 12.dp between dot+timestamp), the `Row` would have had to switch to per-child `Modifier.padding(‚Ä¶)` and lose the arrangement DSL ‚Äî that's a bigger refactor and was correctly avoided here.
- **Belt-and-suspenders preview coverage is two states, not four.** The AC ("with-name promoted, untitled-with-sleeping-dot") names the matrix exactly: one of {named, untitled} √ó one of {awake, sleeping} per preview. The previous file had two previews seeded with `Message` content vs. `null`, which exercised a now-removed axis; the new previews exercise the axes that actually survive (name presence √ó sleep state). Don't add a third preview for "named awake" ‚Äî `ConversationRowPromotedPreview` already covers it.
