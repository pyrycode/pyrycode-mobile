## Patterns established

- **Figma-derived vector drawables go under `res/drawable/`, not `assets/`**, with a single tintable color across all paths so `Icon(tint = colorScheme.*)` works in both themes without per-theme variants. `#FFFFFF` (or any uniform color) is the canonical fill ‚Äî Compose `Icon` replaces it via tint.
- **Atmospheric overlays are `Modifier.drawBehind` on an outer `Box`**, not sibling composables that consume layout space. The content `Column` lives above it in the same `Box`, padded with `systemBarsPadding()`. Center / radius / colors are derived from Figma but expressed as fractions of `size` so the gradient adapts to any preview dimension.
- **Layout dp literals stay inline.** The "no hardcoded values" guardrail bans color (`Color(0x‚Ä¶)`) and typography (`TextStyle(...)`, font sizes / weights / families) literals only ‚Äî `136.dp`, `32.dp`, `28.dp`, `12.dp`, `8.dp`, `16.dp`, `56.dp`, `104.dp`, `20.dp` are fine. Pull them from the Figma payload; ¬±4dp tuning is acceptable.
- **Default `Button` / `TextButton` colors are the contract.** Do not add `ButtonDefaults.buttonColors(...)` overrides ‚Äî the M3 defaults already resolve `primary` / `onPrimary` from the active scheme, which is what the Figma encodes.

## Lessons learned

- **Ticket AC copy vs Figma copy diverged on this one** ‚Äî AC2 quoted headline `"Your Claude sessions, on your phone."` and "supporting paragraph (existing copy preserved)", but Figma node `6:32` had `"Pyrycode Mobile"` as the headline, `"Control Claude sessions on your phone."` as a `titleMedium` tagline, and a longer two-sentence body. The architect followed Figma (the ticket title says "match Figma node 6:32") and flagged the discrepancy. Pattern for future Figma-anchored tickets: when AC and Figma disagree, **Figma is the locked source**; flag in the spec's open questions, follow Figma, let code-review confirm with the PO.
- **`Color.Transparent` is not a hex literal.** It's a `Color` constant, so it passes the "no `Color(0x‚Ä¶)`" guardrail. Useful as the terminal stop of a radial gradient when the gradient should fade into whatever sits below ‚Äî here, the solid `surface` base.
- **Radial-gradient `Brush` doesn't need `remember`.** It's constructed per recomposition from theme-derived `Color` values; allocation is cheap and the lambda inside `drawBehind` runs in the draw phase regardless. The screen has no state changes that would trigger frequent recomposition anyway.
- **Vector drawable conversion from Figma SVG is a hand step.** Figma's `get_design_context` returns SVG; Android `<vector>` is a different XML schema (no `<svg>` root, `pathData` instead of `d`, viewport via `viewportWidth/Height`). Either Android Studio's Vector Asset import, or hand-convert for simple paths. Keep a single uniform fill so `Icon(tint = ‚Ä¶)` controls the color.
