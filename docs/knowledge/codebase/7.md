# #7 — Welcome screen scaffold

First onboarding-flow ticket. Adds a pure stateless `WelcomeScreen` Composable that #8 (NavHost) and #14 (Scanner + browser intent) will wire up.

## Summary

- New package `de/pyryco/mobile/ui/onboarding/` with one file: `WelcomeScreen.kt`.
- Signature is fixed by AC and consumed verbatim downstream:
  `@Composable fun WelcomeScreen(onPaired: () -> Unit, onSetup: () -> Unit)`.
- Layout: hero (logo placeholder + title + subtitle), CTA stack (filled primary + outlined secondary), open-source footer line. Top-level `Surface` + single `Column` with `SpaceBetween`.
- Two `@Preview`s (light + dark) at the Figma `412×892` baseline, both wrap `PyrycodeMobileTheme` with `darkTheme` passed explicitly.

## Files touched

- `app/src/main/java/de/pyryco/mobile/ui/onboarding/WelcomeScreen.kt` — new file, 124 LOC.

## Patterns established

- **`ui/onboarding/` package exists.** Subsequent onboarding screens (pairing, setup) go here.
- **Previews pass `darkTheme` explicitly** to `PyrycodeMobileTheme`. Don't rely on the IDE preview-runner's system setting — it's not deterministic across machines.
- **Stateless onboarding Composables take navigation as `() -> Unit` callbacks**, not a `NavController`. This is what lets routing (#8) and destination wiring (#14) land in separate parallel tickets.
- **Logo placeholder is a plain colored `Box`** (no icon) until real branding lands. Keeps the file dependency-free — `material-icons-extended` is *not* in `libs.versions.toml` and pulling it in for a throwaway asset would be scope creep.

## Lessons learned

- The spec's "Hero `Column` with weight(1f) plus inner `Spacer(weight(1f))` above and below" pattern is how you vertically center a block inside a `SpaceBetween` parent without breaking the SpaceBetween distribution. Future onboarding screens with the same hero/CTA/footer rhythm can reuse it.
- Spacing `dp` literals (24.dp, 32.dp, 16.dp, 12.dp) are **not** banned by the "no hardcoded font sizes/weights/families" AC — that AC targets typography only. Layout dp values are fine inline.
