# #20 — Sleeping-session indicator on `ConversationRow`

Third decoration on `ConversationRow` (after #17 set up the row and #19 added the workspace label). Conversations whose current Claude session is closed render an 8.dp leading status dot in the headline `Row`; active rows are visually unchanged from the #17/#19 baseline.

## Summary

- New field `isSleeping: Boolean = false` appended to `Conversation` (`data/model/Conversation.kt`). Defaulted so the existing constructor sites continue to compile unchanged. The field is **derived, not stored** — Phase 1 stamps it at projection time; Phase 4's real backend will parse it from the conversations endpoint response.
- `FakeConversationRepository.observeConversations` now does the derive: `record.conversation.copy(isSleeping = record.sessions[record.conversation.currentSessionId]?.endedAt != null)`. The repository is the only layer that already holds both halves of the lookup (the conversation and its session map), so the derive lives there — no new repository surface, no new state shape downstream.
- Heuristic: **`currentSession.endedAt != null`**. The default offered in AC #3, picked because `mintNewSession` is the only path that closes a session — it stamps `endedAt = now` on the formerly-current session and writes a new active session with `endedAt = null`. So the heuristic is true exactly when "the current session is closed and no new one has started yet" — the literal data-model definition of sleeping. Clock-independent, monotonic, no flicker. `lastUsedAt` idle-thresholds were rejected because they'd gate UI on test-clock vs. wall-clock.
- One seed channel becomes sleeping: `seed-channel-joi-pilates` (the most stale of the three channels, `lastUsedAt = 2026-05-08`). `seedChannel(...)` grew an optional `currentSessionEndedAt: Instant? = null` parameter; only this seed passes a non-null value (`2026-05-09T03:00:00Z`). Indicator visible at runtime, not only in previews.
- Visual: a leading 8.dp `Box` filled with `MaterialTheme.colorScheme.outline` and `CircleShape`, rendered as the first child of the headline `Row` when `isSleeping == true`. Sits opposite the workspace label (`#19`, trailing edge of the same `Row`); the two decorations don't compete for headline space. The dot is absent when `isSleeping == false` — `Arrangement.spacedBy(8.dp)` only contributes gaps between rendered children, so AC #2 (active rows visually unchanged) holds.
- No `Modifier.alpha` on the `ListItem` itself. The dot alone carries the entire signal — pairing it with row-wide opacity would push the supporting-content preview into a "looks disabled, not tappable" register. Row stays tappable, preview stays legible.
- Preview compaction (AC #4): `ConversationRowWithoutMessagePreview` (the dark preview) now passes `isSleeping = true` and covers three dimensions in one preview (no message × scratch-discussion × sleeping). The light preview's default `isSleeping = false` continues to model the active baseline. Same pattern #19 used — vary existing previews' seed args by dimension instead of padding the file with a third `@Preview`.
- `ChannelListScreenLoadedPreview` got the same one-line treatment: third channel ("scratch ideas") now passes `isSleeping = true` so the list-density view exercises the indicator too. Not required by any AC, but matches the runtime user's experience.
- AC's only build gate is `./gradlew assembleDebug` — passes.

## Patterns established

- **Status fields a real backend will eventually report belong on the model with a default — derived at projection in Phase 1.** Picked over either UI-side computation (would force every caller to thread the signal in) or carrying the raw `Session` through the UI layer (pulls four irrelevant fields just to read `endedAt`). The shape — `val isSleeping: Boolean = false` on `Conversation`, derived in `FakeConversationRepository.observeConversations` — anticipates the Phase 4 wire shape (server stamps the bool directly) without committing to it. The interface contract (`Conversation.isSleeping: Boolean`) is what survives the migration; the Phase 1 derivation is throwaway. Future status-y fields (e.g. `hasUnread`, `isMuted`) should follow the same shape.
- **Defaulting new model fields to their non-meaningful value keeps the construction-site blast radius at zero.** The existing 9 `Conversation(...)` call sites (3 in `FakeConversationRepository` seed factories, 4 in tests, 1 in `ChannelListScreen` preview, 1 in `ConversationRow.kt` preview) compiled untouched. Only the call sites that needed to assert the meaningful value (the two preview sites) edited, passing `isSleeping = true` as a named arg. Compare to `name: String?` (no default — `null` is meaningful and every caller must decide). For booleans whose "off" state is uninteresting and represents the past behaviour, `= false` is the right discipline.
- **The repository projection is the one place a derive that joins entity + child collection should live.** `observeConversations` already had both `record.conversation` and `record.sessions` in scope; the derive is one line in `Flow.map`. No new repository method, no new state shape, no new query path. Two-line change in the projection beats a structural rework of the downstream `UiState`.
- **Defensive `?.` short-circuit beats explicit `let { ... } ?: false` when reading a possibly-missing nested field.** `current?.endedAt != null` is concise idiomatic Kotlin; `null != null` is `false`, which is the right behaviour when `record.sessions[currentSessionId]` is `null` (a programming error — but the projection should never crash the conversations Flow on it).
- **Visual decorations prefer `Box(Modifier.size(N.dp).background(color, CircleShape))` over `Icon` when one glyph would otherwise pull in `material-icons-extended`.** That library bundles thousands of icons (multi-MB APK cost) and isn't on the classpath today; adding it for a single dot inverts the "no chips, no surfaces, no extra widgets when a simpler primitive suffices" discipline #17/#19 established. A `Box`-and-shape primitive is height-flat (no layout shift versus other rows) and zero-dep. The same primitive will fit the next "show a small status mark" need too.
- **Headline decorations split by edge: leading vs. trailing.** #19's workspace label took the trailing edge of the headline `Row`; this ticket takes the leading edge. The two decorations are now diametric and don't compete for the same headline real estate. A future fourth decoration on the headline should think about which edge before adding a third spacedBy gap inside the existing `Row`.
- **Decoration semantic-color is `colorScheme.outline` for "dormant / inactive".** Not `error` (alarming), not `surfaceVariant` (invisible against the surface), not `primary` (would read as a call-to-action). M3's `outline` is the role for inactive borders/dividers — exactly the register for "this conversation is dormant". Colour-scheme-aware (auto-adjusts for dark mode); no `Color(0x...)` literal.
- **Preview compaction by varying an existing preview's seed args, not by adding a third `@Preview`.** Same shape #19 used: `ConversationRowWithoutMessagePreview` already covered (no message × scratch-discussion × dark theme); flipping its seed's `isSleeping = true` adds a fourth dimension to the same preview without bloating the file. Two `@Preview` functions still cover everything that matters. When a dimension-fifth needs adding, this trick is still in the toolkit.

## Lessons learned

- **`Modifier.background(color = …, shape = CircleShape)` is the single-modifier shape-fill idiom — no `Surface`, no `Card`, no `clip` ceremony.** The `background` overload that takes a `Shape` clips its own background; you don't need a separate `Modifier.clip(CircleShape)` first. Three modifiers (`size`, `background(color, shape)`) draw a coloured circle. Compare to a `Surface(shape = CircleShape, color = …) { Spacer(...) }` chain which is two extra widgets for the same pixels.
- **`Arrangement.spacedBy(N.dp)` only spaces between *rendered* children, so conditionally emitting children is layout-safe.** Reaffirms #19's lesson — the `if (conversation.isSleeping) Box(...)` inside the headline `Row` body emits zero or one child; when zero, no leading 8.dp gap appears, and the active row's headline starts flush with the name (matching the #17/#19 baseline). No `Modifier.padding` arithmetic, no second `Row` variant.
- **`material-icons-core` (the always-on icon set) does not contain a "sleeping" / "moon" glyph.** The closest active-classpath candidate is `Icons.Outlined.Schedule` (a clock face), but a clock reads as "duration / time" rather than "asleep" — semantically muddier than a dot-with-low-contrast-tint. Any future "I want a glyph for X status" check should grep `androidx.compose.material.icons.Icons.Outlined` first, and fall back to a `Box`-shape primitive before reaching for `material-icons-extended`.
- **Phase-1 derive vs. Phase-4 wire field is a deliberate seam, not a hack.** The `Conversation.isSleeping` field's *interface shape* is what the Phase 4 backend will produce; the Phase-1 derivation in `FakeConversationRepository.observeConversations` is implementation-detail that goes away when the Ktor-backed impl lands. The downstream contract (UI reads `conversation.isSleeping: Boolean`) doesn't move. Naming this seam in the ticket file so the Phase 4 work doesn't rediscover it.
- **Don't pair the dot with row-wide `Modifier.alpha(0.6f)` — two signals would read as "disabled / not tappable".** Considered and rejected. Sleeping conversations are still tappable (they re-attach on next message), so the row must not look greyed-out. The dot is the entire signal; the supporting-content preview text stays at full opacity. Worth pinning because the "fade the whole row" reflex is wrong for a tappable status.
- **Defensive null in the projection (`current?.endedAt != null`) intentionally projects as `isSleeping = false`, not as a thrown invariant violation.** If `record.sessions[currentSessionId]` were ever `null` — a programming error guarded by the seed factories and `mintNewSession` — the projection would still produce a valid `Conversation` rather than crash the upstream `Flow`. The seed-invariant test (`seededChannels_haveTwoSessionsInHistory_endingWithCurrentSessionId`) catches construction-side breaks; the projection doesn't need a second crash path.
